## Qualitative Read

- The model did not follow the JSON response format on any example.
- Raw generations were repetitive degenerate text rather than coherent answers or critiques.
- The collapse is uniform across `false_premise`, `conflicting_evidence`, and `answerable_control`, which suggests instruction-following failure before any task-specific reasoning.
- A compact prompt variant on `day1_quick` produced the same total abstention collapse, so weak-model failure is not explained by prompt verbosity alone.
- `Qwen2.5-0.5B-Instruct` crosses the first capability threshold: it gives nontrivial action distributions and reaches `0.625` action accuracy on `conflicting_evidence`.
- The main failure mode shifts from total collapse to policy mismatch. On `day1_dev`, `false_premise` stays at `0.0` action accuracy while `conflicting_evidence` is much better, so the benchmark is separating "challenge a bad premise" from "notice missing evidence."
- JSON formatting is still weak even once reasoning improves: parse rate rises above zero, but only to `0.0667` on `day1_dev`, so evaluation must still tolerate fallback parsing for small models.
- The model is still poor on direct answering (`answer_em=0.0`), so improved action selection is not yet enough for end-to-end helpfulness.
- `Qwen2.5-Coder-7B-Instruct` is the first baseline that nearly saturates output formatting (`json_parse_rate=0.9417`) while materially improving the hard slice (`false_premise` reaches `0.45` action accuracy on `day1_dev`).
- Even the stronger 7B model keeps a large gap on `answerable_control` (`0.35` action accuracy on `day1_dev`), which suggests the benchmark is also stressing completion willingness and not only premise critique.
- A quick audit of the 7B model's `answerable_control` misses shows a strong pattern: `26/26` action errors come from `PCBench`, `21/26` are `abstain`, and most are LaTeX-heavy prompts without a partial-solution scaffold.
- The stricter `answer_em` metric is too brittle for short-answer factual QA because full-sentence correct answers are marked wrong when they do not exactly match the gold span. We now log `answer_contains_rate` alongside `answer_em` to separate genuine answer failure from evaluation string mismatch.
- Prompt-only fixes can game the wrong metric. The `self_contained` ablation raises `action_accuracy` dramatically by turning nearly all answerable controls into `answer`, but it also degrades `false_premise` calibration and makes overall utility substantially worse.
- The utility metric is doing real work here: without it, the `self_contained` ablation would look like a major improvement even though it is unsafe on the benchmark's hardest slice.
- The first stale-premise seed draft exposed a benchmark-design failure, not a model failure. With no retrieved passages, even the strong 7B model abstained on all `7/7` items despite perfect JSON formatting.
- The stale slice becomes meaningful once lightly grounded. Adding two short evidence passages per example raises `Qwen2.5-Coder-7B-Instruct` from `0.0` to `0.8571` action accuracy on the same seven stale items, while `Qwen2.5-0.5B-Instruct` still stays at `0.0`.
- The stale slice now differentiates capability in a paper-useful way: the 7B model mostly challenges correctly, but the 0.5B model still alternates between `ask` and unsupported `answer`, so the effect is not just a formatting artifact.
- The grounded stale slice still works inside the mixed benchmark. On `day1_quick_plus_stale`, `Qwen2.5-Coder-7B-Instruct` reaches `1.0` stale-premise action accuracy (`4/4`), while `Qwen2.5-0.5B-Instruct` remains at `0.0`, so the effect is not limited to an isolated probe file.
- The first DeepSeek runs uncovered a protocol bug in our reasoning setup, not a model property. Omitting the system prompt without inlining the benchmark instructions into the lone user message guaranteed free-form reasoning text and near-total fallback abstention.
- After fixing that user-only protocol and later repairing the fallback parser to prioritize explicit final-response cues, `DeepSeek-R1-Distill-Qwen-1.5B` still does not emit usable JSON, but it becomes a scientifically meaningful baseline: it answers many more items, reaches `0.65` action accuracy on `conflicting_evidence`, and shows clear over-answering on `false_premise` and especially `stale_premise`.
- The `compact` user-only prompt does not solve formatting, but it does change calibration in a consistent direction on both quick and dev: fewer direct answers on defective-premise items, better utility, and worse answerability on clean controls.
- The corrected reasoning failure mode is not “too cautious” but “wrongly explanatory.” On grounded stale items, the model often reads the update correctly and still decides the premise is acceptable, then produces an answer-style explanation instead of a direct correction.

## Immediate Implication

- `135M` is useful as a floor baseline showing the task is not trivial.
- `Qwen2.5-0.5B-Instruct` is the first informative baseline: the task is no longer dominated by instruction collapse, and the slice structure now exposes distinct policy failures.
- `Qwen2.5-Coder-7B-Instruct` shows that stronger models can recover both JSON structure and part of the `challenge` policy without changing the prompt, so the benchmark signal is not capped by formatting collapse.
- The next informative checkpoint is now a reasoning-oriented model that may be more willing to challenge bad premises while also reducing abstention on clean answerable items.
- Prompt robustness matters: the benchmark should report when failures come from format-following collapse versus action-selection errors.
- The compact-prompt ablation is worth retaining in the paper appendix because it rules out a cheap “prompt too long” explanation for the weakest baseline.
- Reporting needs two answer metrics: strict `answer_em` for exactness and `answer_contains_rate` for short-answer factual coverage.
- The benchmark should keep the conservative main prompt as the primary protocol. More aggressive prompts such as `self_contained` are useful diagnostics, but they change the calibration profile too much to serve as the main evaluation setting.
- The stale-premise slice should not be evaluated in a zero-passage form. That variant mainly measures whether the prompt licenses world-knowledge correction without support, which is a different question from calibrated challenge under grounded outdated premises.
- A grounded stale slice is promising enough to keep: it adds a third defect type that is neither identical to `false_premise` nor reducible to `conflicting_evidence`, and it already shows a sharp 0.5B vs 7B gap.
- The quick four-slice comparison is now paper-usable: `0.5B` vs `7B` diverges sharply on both `false_premise` and `stale_premise`, while `conflicting_evidence` remains relatively easier, which is exactly the kind of slice-structured story we need.
- The corrected DeepSeek result is weaker than the `0.5B` instruct model on overall utility, but it is more diagnostically interesting: after parser-fix reparse it partially recovers challenge behavior while also over-answering defective-premise slices, which is closer to the paper's intended reasoning-risk story than the earlier abstention-collapse artifact.
- The DeepSeek `compact` ablation is worth keeping because it shows a real calibration lever: prompt compression can trade answerability for safety even when JSON parse rate stays flat.
